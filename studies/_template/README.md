# New Study Template

复制这个目录来新建一个算法验证子目录。

## 最少保留的结构

- `README.md`
- `run_study.m`
- `outputs/`

## 建议 README 固定写法

新建一个实验目录时，README 至少写清楚下面几项：

- 这个实验要验证什么
- 主入口是什么
- 需要哪些前置条件
- 常用参数有哪些
- 给 2 到 4 个可直接复制的示例命令
- 输出文件会写到哪里
- 这个实验适合放什么，不适合放什么

## 建议入口写法

推荐 `run_study.m` 里固定保留下面的基本动作：

1. 找到当前 `study_dir`
2. 创建 `outputs/`
3. 调用 `init_project_paths`
4. 切到 `outputs/`
5. 调用真正的实验入口

这样每个实验的运行习惯都一致，后续维护成本最低。

## 建议参数组织方式

如果参数不多，可以直接让 `run_study(cfg)` 接受结构体。

如果参数很多，建议：

- 在当前实验目录新增一个 `default_config.m`
- 主入口里先取默认值，再应用用户覆盖

## 示例模板

可以直接照下面这种形式写：

```matlab
cd studies/my_new_study
init_project_paths
result = run_study;
```

带参数运行：

```matlab
cfg = struct('param_a', 1.0, 'param_b', 32, 'plot_results', true);
result = run_study(cfg);
```

## 推荐迁移顺序

1. 先把旧入口脚本接进 `run_study.m`
2. 确认默认产物都写进 `outputs/`
3. 再把这个实验特有的后处理脚本一起搬进来
4. 如果某段逻辑被多个实验复用，再抽回公共层

## 判断是否应该抽成公共库

满足下面任意一条，就应该从 study 目录抽出去：

- 两个以上实验都依赖
- 和具体实验名称无关
- 主要功能是参数初始化、模型构建、结果导出、通用评价

## 判断是否应该留在 study 目录

满足下面任意一条，就应该留在当前实验目录：

- 只服务一个实验链路
- 文件名里明显带有当前实验语义
- 主要产物是当前实验自己的图表和结果文件

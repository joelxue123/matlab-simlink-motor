# Motor Control Modules

This directory is the team-facing reuse entry for the motor-control MBD work.
It does not move or rewrite the milestone models. Instead, it gathers the
already validated modules behind one setup script, one library build script,
one test entry, and shared documentation.

The shared data dictionary is:

```text
motor_control_interface.sldd
```

It is generated by:

```text
build_motor_control_interface_dictionary.m
```

## Quick Start

From the repository root:

```bash
matlab -batch "run('motor_control_modules/setup_motor_control_modules.m'); run('motor_control_modules/build_motor_control_interface_dictionary.m'); run('motor_control_modules/build_motor_control_module_library.m')"
```

Then open:

```text
motor_control_modules/motor_control_lib.slx
```

To show the library in the Simulink Library Browser, run:

```matlab
run('motor_control_modules/setup_motor_control_modules.m')
sl_refresh_customizations
```

The browser entry is:

```text
Motor Control Modules
```

Team members should reuse blocks from this library instead of copying
subsystems out of demo models.

Copy/paste from `motor_control_lib.slx` is acceptable if the pasted block keeps
its library link. Check a selected pasted block with:

```matlab
get_param(gcb, 'LinkStatus')
get_param(gcb, 'ReferenceBlock')
```

Expected:

```text
LinkStatus     = resolved
ReferenceBlock = motor_control_lib/<ModuleName>
```

## Reusable Modules

Current reusable candidates:

```text
SpeedPiStep
CurrentPiStep
DqToAbcDutyStep
MotorClarkeParkStep
OpenLoopCommand
DeadtimeCompensationStep
```

Recommended meaning:

- `SpeedPiStep`: production controller candidate, outputs `iq_ref`.
- `CurrentPiStep`: production controller candidate, outputs `vd_ref/vq_ref`.
- `DqToAbcDutyStep`: production controller candidate, outputs `[0, 1]` duty.
- `MotorClarkeParkStep`: production controller candidate, converts phase current
  feedback to dq current.
- `OpenLoopCommand`: startup/debug candidate; keep it separate from the normal
  closed-loop controller path.
- `DeadtimeCompensationStep`: production PWM adapter candidate, compensates
  duty using dq-synthesized phase-current polarity.

## Validation

Run the module regression entry:

```bash
matlab -batch "run('motor_control_modules/tests/run_all_module_smoke_tests.m')"
```

This calls the existing milestone tests instead of inventing new duplicate
tests.

## Documentation

- `docs/module_contracts.md` records interfaces, units, sample times, and reuse
  rules.
- `docs/reuse_integration_guide.md` explains how another project should consume
  the modules at Simulink level and C level.

## Reuse Rule

Do not copy algorithm blocks out of demo models. Reuse one of these forms:

```text
Simulink Library block
Model Reference
Protected Model
Generated C + platform adapter
```

The generated control code must stay platform independent. TI, ST, NXP, and
AUTOSAR integrations belong in platform adapters, not inside the controller
core.

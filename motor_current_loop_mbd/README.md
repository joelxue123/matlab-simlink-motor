# Motor Current Loop MBD

This directory contains the first integrated current-loop motor-control
simulation milestone.

The test command is intentionally simple:

```bash
matlab -batch "run('motor_current_loop_mbd/run_current_loop_id_step_smoke_test.m')"
```

## Architecture

```text
id_ref step, iq_ref = 0
  -> current_pi_input_t
  -> CurrentPiStep
  -> current_pi_output_t
  -> dq voltage command adapter
  -> DqToAbcDutyStep
  -> phase_duty_t
  -> Rate Transition 25us
  -> Average-Value Inverter
  -> Surface Mount PMSM
  -> plant_feedback_t
  -> Rate Transition 50us
  -> motor_t
  -> MotorClarkeParkStep
  -> motor_dq_t
  -> current_pi_input_t feedback
```

This is an integration model, not a single giant algorithm model. The reusable
controller-facing subsystems are:

- `CurrentPiStep`
- `DqToAbcDutyStep`
- `MotorClarkeParkStep`

The inverter and PMSM plant are simulation environment blocks.

## ID-Step Milestone

The first closed current-loop test uses:

```text
id_ref = 0 A -> 3 A step
iq_ref = 0 A
```

For a surface PMSM, `iq_ref = 0` means the test should not produce meaningful
average electromagnetic torque. This lets the d-axis current loop be tested
without speed-loop or motion effects.

Current smoke-test result:

```text
id_ref final  = 3 A
id_meas final = 3 A
iq_meas range = [0, 0] A
vd_ref range  = [0, 4.15697] V
vq_ref range  = [0, 0] V
wm max abs    = 0 rad/s
duty range    = [0.454151, 0.545849]
Current-loop id-step smoke test passed.
```

## Sample Times

```text
Controller: 50 us
Inverter/plant: 25 us
```

The model uses Rate Transition blocks at both important boundaries:

- controller duty output to 25 us inverter input
- 25 us plant feedback to 50 us controller input

## Files

- `build_motor_current_loop_model.m` creates the `.sldd` and `.slx`.
- `run_current_loop_id_step_smoke_test.m` builds and simulates the ID-step
  current-loop test.
- `generate_current_loop_code.m` builds the model and runs Embedded Coder code
  generation.
- `motor_current_loop_interface.sldd` is generated by the build script.

## Code Generation

Run:

```bash
matlab -batch "run('motor_current_loop_mbd/generate_current_loop_code.m')"
```

Generated code is written to:

```text
motor_current_loop_mbd/motor_current_loop_model_ert_rtw/
```

Important generated algorithm files:

```text
CurrentPiStep.c/.h
DqToAbcDutyStep.c/.h
MotorClarkeParkStep.c/.h
motor_current_loop_types.h
```

## Design Notes

- All controller-facing types are float-first `Simulink.AliasType('single')`.
- The model keeps `[0, 1]` duty semantics for the Average-Value Inverter.
- The first test deliberately avoids speed loop and torque-producing q-axis
  current.
- The next useful milestone is an `iq_ref` step with rotor locked or a carefully
  controlled mechanical setup, then dq voltage vector limiting before wider FOC
  tests.

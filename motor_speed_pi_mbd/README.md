# Motor Speed PI MBD

This directory contains the reusable speed-loop PI controller module for the
motor MBD workflow.

The module is implemented with Simulink primitive blocks instead of a MATLAB
Function block or a black-box PID Controller block.

## Architecture

```text
speed_pi_input_t
  -> SpeedPiStep
  -> speed_pi_output_t
```

Input bus:

```text
speed_pi_input_t {
  wm_ref
  wm_meas
  iq_limit
}
```

Output bus:

```text
speed_pi_output_t {
  iq_ref
}
```

The controller implements:

```text
error = wm_ref - wm_meas
iq_pre_sat = Kp_speed * error + integrator
iq_ref = clamp(iq_pre_sat, -iq_limit, iq_limit)
integrator_next = integrator + Ts * (Ki_speed * error + Kaw_speed * (iq_ref - iq_pre_sat))
```

## Files

- `build_speed_pi_model.m` creates `speed_pi_interface.sldd` and
  `speed_pi_model.slx`.
- `run_speed_pi_smoke_test.m` builds and simulates the model with a simple
  mechanical plant harness.
- `generate_speed_pi_code.m` builds the model and runs Embedded Coder code
  generation.
- `speed_pi_interface.sldd` is generated by the build script.

## Run

From repository root:

```bash
matlab -batch "run('motor_speed_pi_mbd/run_speed_pi_smoke_test.m')"
```

Current smoke test result:

```text
wm_ref final  = 41.8879 rad/s
wm_meas final = 41.8879 rad/s
speed error   = 0 rad/s
iq_ref range  = [-0.765256, 15] A
iq_limit      = 15 A
Speed PI smoke test passed.
```

The first response saturates `iq_ref` at the positive current limit, then
settles to the requested mechanical speed.

```bash

Generated code is written to:

                             P_SpeedPiStep_T *localP);

                        P_SpeedPiStep_T *localP);
```

## Design Notes

- The current green-joint mainline speed loop runs at `100us`, slower than
  the `50us` current loop but fast enough to avoid the old 500us scheduling
  bottleneck.
- The output `iq_ref` is the command for the q-axis current loop.
- Parameters `Kp_speed`, `Ki_speed`, `Kaw_speed`, and `IqLimitDefault` live in
  the `.sldd`.
- Controller-facing scalar types are `Simulink.AliasType('single')`.
- Baseline gains are derived from `average-inverter/motor_control_params.m`.
- The next integration milestone is:

```text
SpeedPiStep -> CurrentPiStep -> DqToAbcDutyStep -> Average Inverter -> PMSM
```

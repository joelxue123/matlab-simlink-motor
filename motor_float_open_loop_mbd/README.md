# Motor Float Open-Loop MBD

This directory is the first version of a new motor simulation architecture
inspired by `average-inverter`, rebuilt with the modern MBD workflow.

The current milestone is open-loop motor testing with float controller
interfaces.

## Architecture

```text
OpenLoopCommand
  -> open_loop_cmd_t
  -> DqToAbcDutyStep
  -> phase_duty_t
  -> Average-Value Inverter
  -> Surface Mount PMSM
  -> plant_feedback_t
```

The controller-facing interfaces use `single` through `.sldd` type aliases:

```text
T_MotorFloat
T_MotorVoltage
T_MotorAngle
T_MotorCurrent
T_MotorSpeed
T_MotorTorque
T_DutyRatio
```

The physical plant block is a Motor Control Blockset library block. The model
uses explicit Data Type Conversion blocks at the plant boundary so controller
interfaces stay float while the library plant can use the data types it expects
internally.

The Average-Value Inverter input boundary runs at the plant sample time:

```text
simcfg.Ts_plant = 25e-6
```

`PhaseDuty_RateTransition_25us` and `Vdc_RateTransition_25us` explicitly define
the 25 us task boundary before the duty and DC bus inputs enter the
Average-Value Inverter.

## Files

- `build_motor_float_open_loop_model.m` creates `motor_float_interface.sldd`
  and `motor_float_open_loop_model.slx`.
- `run_open_loop_smoke_test.m` builds and simulates the model, then checks that
  feedback values are finite.
- `motor_float_interface.sldd` is generated by the build script.

## Run

From repository root:

```bash
matlab -batch "run('motor_float_open_loop_mbd/run_open_loop_smoke_test.m')"
```

Current smoke test result:

```text
Open-loop smoke test passed.
```

Expected warnings:

```text
Parameter precision loss ... single
```

These warnings are acceptable for this float-first milestone. They indicate
that constants such as `2*pi`, `sqrt(3)/2`, and `1/3` are represented as
single precision.

## Design Notes

- This is not yet a closed-loop FOC controller.
- The first goal is a clean, reusable motor simulation skeleton.
- The open-loop angle generator uses discrete accumulation, which is closer to
  an MCU implementation than `theta = omega*t` from a Clock block.
- `phase_duty_t` carries `[0, 1]` duty ratios for Average-Value Inverter.
- `DqToAbcDutyStep` follows the `speedloop_kf_test` convention:
  `duty = v_svpwm / Vdc + 0.5`.
- The Average-Value Inverter boundary is explicitly sampled at 25 us using
  Rate Transition blocks.
- Future fixed-point work should use the same `.sldd` pattern and replace
  selected AliasType entries with `Simulink.NumericType`.
